## Список реализованных маршрутов с описанием

```csharp
// Маршрут 1: Главная страница
app.MapGet("/", () => "Добро пожаловать на сервер!");
// Маршрут 2: О нас
app.MapGet("/about", () => "Это мой первый ASP.NET Core сервер");
// Маршрут 3: Текущее время о нас
app.MapGet("/time", () => $"Время на сервере: {DateTime.Now}");
// Маршрут 4: Параметр пути
app.MapGet("/hello/{name}", (string name) => $"Привет, {name}!");
// Маршрут 5: Возвращаем JSON - обЪект
app.MapGet("/student", () => new {
    Name = "Rinat Abdulin",
    Group = "ISP-233",
    Year = 3,
    IsActive = true
});
// Маршрут 6: JSON - массив
app.MapGet("/subjects", () => new[] {
    "RPM",
    "RMP",
    "ISRPO",
    "SP",
});
// Маршрут 7: JSON с использованием класса
app.MapGet("/product/{id}", (int id) => new Product(
    Id: id,
    Name: $"Товар #{id}",
    Price: id * 99.99m,
    InStock: id % 2 == 0
));
```

## Вопросы и ответы

### 1. Чем Minimal API отличается от MVC в ASP.NET Core?

| Характеристика | Minimal API | MVC |
|---|---|---|
| **Объём кода** | Минимальный, меньше шаблонов | Требует контроллеры, модели, представления |
| **Структура** | Всё в одном файле (Program.cs) | Разделение на Controllers, Models, Views |
| **Сложность** | Простой, для небольших API | Мощный, для крупных веб-приложений |
| **Фильтры** | Ограниченная поддержка | Полная поддержка фильтров |
| **Модельное связывание** | Простое | Расширенное с атрибутами |
| **Подходит для** | Микросервисы, простые API | Сложные веб-приложения, SPA |

### 2. Что произойдёт, если поставить `app.MapGet(...)` до `app.Use(...)`?

Middleware (`app.Use()`) **не будет выполняться** для этого маршрута, так как порядок важен:

```csharp
// Неправильный порядок
app.MapGet("/", () => "Hello"); // Этот маршрут НЕ будет обработан middleware
app.Use(async (ctx, next) => {
    // Этот код не выполнится для "/"
    await next();
});
// Правильный порядок
app.Use(async (ctx, next) => {
    // Этот middleware выполнится для всех запросов
    Console.WriteLine("Логирование...");
    await next();
});
app.MapGet("/", () => "Hello"); // Маршрут после middleware
```

### 3. Почему ASP.NET Core преобразует PascalCase в camelCase при сериализации JSON?

Причина: Это стандарт JavaScript/JSON (camelCase), в то время как C# использует PascalCase.

```csharp
builder.Services.Configure<JsonOptions>(options => 
    options.SerializerOptions.PropertyNamingPolicy = null
);
```

### 4. Что означает код ответа HTTP 401? А 404? А 200?

| Код | Название | Значение | Когда возвращается |
|-----|----------|----------|-------------------|
| **200** | OK | Запрос успешно выполнен | Данные найдены и возвращены клиенту |
| **401** | Unauthorized | Пользователь не авторизован | Требуется аутентификация (например, неверный ключ API) |
| **404** | Not Found | Ресурс не найден | Маршрут или файл не существует на сервере |

**Дополнительно:**
- **401** vs **403**: 401 - не авторизован (нужен логин), 403 - доступ запрещён (вошёл, но прав нет)
- **200** может быть пустым телом ответа
- **404** часто возвращается через `Results.NotFound()`

### 5. Чем `dotnet run` отличается от `dotnet watch run`?

| Команда | Описание | Когда использовать |
|---------|----------|-------------------|
| `dotnet run` | Однократная компиляция и запуск | При финальном тестировании, продакшене |
| `dotnet watch run` | Запуск с **автоматической перезагрузкой** при изменении файлов | При разработке (DEV-режим) |

**Пример работы `dotnet watch run`:**
```bash
dotnet watch run
# > Запуск сервера...
# > Изменили код → watch автоматически пересобирает
# > файлы изменены. Перезагрузка...
# > Сервер перезапущен
```
